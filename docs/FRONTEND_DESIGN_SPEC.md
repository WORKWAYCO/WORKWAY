# WORKWAY Frontend Design Specification

**Philosophy**: *Zuhandenheit in interfaceâ€”the UI recedes, the workflow emerges.*

---

## Design Principles

| Principle | Implementation |
|-----------|----------------|
| **Weniger, aber besser** | Every element earns its place |
| **Progressive disclosure** | Complexity revealed on demand |
| **Intent over mechanism** | Show what, not how |
| **Trust through transparency** | Attribution always visible |
| **Honest metrics** | Only show data that exists |

---

## Early-Stage vs. Mature Marketplace

This spec includes two states:

| Element | Early Stage (Now) | Mature (With Data) |
|---------|-------------------|-------------------|
| Fork counts | Hidden | Shown when > 0 |
| Install counts | Hidden | Shown when > 10 |
| Reviews/ratings | Hidden | Shown when > 3 reviews |
| "Most Forked" filter | Hidden | Shown when forks exist |
| Developer earnings | Capability described | Actual numbers shown |
| Success stories | Generic/aspirational | Real testimonials |

**Rule**: Never show "0" or placeholder data. Either show real numbers or describe the capability.
